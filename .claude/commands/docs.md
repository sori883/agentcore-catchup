# ドキュメント参照コマンド

プロジェクトのドキュメントを参照・確認するコマンドです。

## 使用方法

```
/docs [ドキュメント種別]
```

### ドキュメント種別

| 種別 | ファイル | 説明 |
|------|----------|------|
| `spec` | `docs/SPEC.md` | 機能仕様書（機能一覧、詳細仕様、キーバインド） |
| `arch` | `docs/ARCHITECTURE.md` | アーキテクチャ設計（構成、技術スタック、設計パターン） |
| `changelog` | `docs/CHANGELOG.md` | 変更履歴（バージョン毎の変更内容） |
| `all` | 全ドキュメント | 全ドキュメントの概要を表示 |

### 使用例

```
/docs spec        # 仕様書を参照
/docs arch        # アーキテクチャを参照
/docs changelog   # 変更履歴を参照
/docs all         # 全ドキュメントの概要
```

## 実行手順

### 1. 指定ドキュメントの読み込み

```
spec → docs/SPEC.md を読み込み
arch → docs/ARCHITECTURE.md を読み込み
changelog → docs/CHANGELOG.md を読み込み
all → 全ドキュメントの目次を表示
```

### 2. 内容の要約表示

ドキュメントの内容を以下の形式で要約：

**SPEC.md（仕様書）の場合：**
- 実装済み機能一覧
- 未実装機能一覧
- 主要なキーバインド

**ARCHITECTURE.md（アーキテクチャ）の場合：**
- ディレクトリ構成
- 技術スタック
- 主要なモジュール

**CHANGELOG.md（変更履歴）の場合：**
- 最新バージョンの変更内容
- 関連Issue/PR

## ドキュメントの参照タイミング

| シーン | 参照するドキュメント |
|--------|---------------------|
| 新機能の設計時 | `/docs spec` で既存仕様を確認 |
| コード構造の理解 | `/docs arch` でアーキテクチャを確認 |
| PRレビュー時 | `/docs changelog` で変更履歴を確認 |
| 機能追加前 | `/docs all` で全体像を把握 |

## 関連コマンド

- `/update-spec` - 仕様書を更新
- `/create-issue` - Issue作成
